The System Metrics Tab

The System Metrics tab provides a more detailed graphical view of system metrics. This page includes detailed line graphs that highlight items such as active queries, memory usage, CPU usage, swap page usage, disk I/O, and network throughput.

You can export the data from any graph or table in the Command Center UI by clicking the Export icon in the upper right. See "Exporting Data" for more details.

Click the Show / Hide Charts to specify which graphs are displayed or hidden on this page.

Viewing All Metrics

The default view for the System Metrics tab is the All Metrics view, which charts data aggregated across all segments in the database array for a period of one hour. To modify the time period displayed in the graphs, select one of the following options in the Duration pull-down menu:

  • 1 hour (default)
  • 6 hours
  • 24 hours
  • 7 days
  • 2 weeks
  • 3 months
  • 1 year

Hovering the mouse cursor on the various points in the line graphics opens a small window that displays the exact numeric metric value, as well as other details about the specific point on the graph. In the example below, this is the CPU usage by user and by system.

Clicking on any point a line graph opens the System Metrics > Details (by metric) view.

Viewing Detailed Information by Metric

To view detailed information about a metric, you can either click on a point in the summary graphs, or select the Details (By Metric) view. For each metric type, there is a generated view that displays a line graph in the upper portion of the page. This information is the average value for that metric across the entire cluster, over time. If you click on a specific point in time on this chart, the table in the lower part of the screen displays the data for the selected metric type (CPU Usage, Memory, etc) for each host at that time.

If you are viewing metrics for Active Queries, the table in the lower part of the screen displays data about all queries on the Database cluster at the moment in time you select, not by host.

Hovering the mouse cursor over any line graph displays exact data for that metric in a popup window. Clicking on the any point in the line graph lists textual data in the Detailed Data (bottom) area.

To view detailed information for different metrics, select any of the following metric types from the pull-down menu in the top left portion of the Detail (By Metric) view:

  • Active Queries
  • Memory
  • CPU
  • Load
  • Swap (in use)
  • Swap (pages in/out)
  • Disk I/O (ops/s)
  • Disk I/O (MB/s)
  • Network (pkts/s)
  • Network (MB/s)

When viewing the Network metrics (both pkts/s and MB/S), you have the option to view the data in either a line graph (Smooth) format or a bar graph (Bars) format. Click the appropriate radio button at the top right of the graph to switch between these formats.
Network metrics include traffic over all NICs (network interface cards); internal interconnect and administrative.

Overlaying Graphs:

Use the Overlay dropdown menu to overlay the graph for one metric over the graph of another metric.

Viewing Realtime Metrics by Server 

The Realtime (By Server) view displays metrics by server, with a separate bar graph for each server in the array. Hovering the mouse cursor over the end point of these bar graphs displays a small window with the exact numeric value for the metric, as shown below. This screen gives you a quick overview of the performance of your system, you can quickly recognize issues such as data skew where one of the segment hosts has disproportionate CPU and/or memory usage.

You can sort by clicking on any of the column headers, clicking a second time changes the direction of the sort. An arrow in the column header indicates that this is the current sort column.

For each server, the following information is displayed:

  • CPU (User/System/Idle)
  • Memory (Used/Free)
  • Disk read rate (MB/s)
  • Disk write rate (MB/s)
  • Net read rate (MB/s)
  • Net write rate (MB/s)

Viewing Historical System Metrics

With a default quantum setting of 15 seconds, Command Center agents collects very large amounts of historical system data. To view meaningful groupings of system data records, you can view the data by selected time intervals with Command Center. You can analyze this data to identify trends in the performance of the Pivotal Greenplum Database system over time.

The All Metrics view in the System Metrics tab displays graphs in intervals up to a year in length, and you can use the Duration pull-down menu change interval. To view system metrics for time periods beyond a year, query the Command Center database using SQL commands as described in the Pivotal Greenplum Command Center Administration Guide.
