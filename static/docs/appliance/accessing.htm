Accessing Pivotal Greenplum Command Center

There are three possible levels of user for Greenplum Command Center.

Superuser. This user can use all GPCC's features; including viewing information for all database queries as well as system metrics and administrative tasks.

Operator. This user is limited to viewing their own database queries and also has limited access to administrative tasks. Note that the Operator role needs to be created and users assigned to that role. Details are provided in the Greenplum Command Center 1.3 Administrator Guide.

Regular users. Regular users are limited to viewing their own database queries and do not have access to Administrative tab.

 

To access Greenplum Command Center, open a supported browser and navigate to the login page. For example, if you are using the default web server port of 28080 to connect with SSL, use this address:

https://monitor_console_hostname:28080

On the login page, enter your credentials and click Login. This opens the Command Center's Dashboard tab.

Note: If configured, you can switch to Multi-Cluster view, to view the status of multiple clusters on one page.  See "Multi-Cluster Support" for more details.

The Dashboard tab provides a graphical system snapshot and a summary view of active queries. Click here for information about this tab.

Supported Browsers

Pivotal Greenplum Command Center user interface is a Rich Internet Application (RIA) requiring Adobe Flash 11.0 or higher. This allows it to run on a variety of modern web browsers. For the complete list of supported browsers, refer to Adobe's website.

Pivotal Greenplum Command Center uses the lighttpd web server. The Command Center UI is typically installed on the Greenplum Database master host, but can be installed on any host with network access to the master host. The Command Center database which stores data for the console, is distributed across the master and the segment hosts of the Greenplum Database (or Database Module in the case of a DCA hosted Command Center). The default web server port is 28080. For more information about the web server, click here .
