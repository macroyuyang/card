The Dashboard Tab

The Dashboard tab displays after you log into Pivotal Greenplum Command Center. The Dashboard tab provides a quick summary view of the current system performance, workload, health, alerts/dial-homes.

Note that some of these features will only be available to you are running Pivotal Greenplum Command Center on an EMC Data Computing Appliance (DCA); for example, dial-homes.

You can export the data from any graph or table in the Command Center UI by clicking the Export icon in the upper right. See "Exporting Data" for more details.

Click the Show / Hide Charts to specify which graphs are displayed or hidden on this page.

Status

The Greenplum Status section of the dashboard gives you a quick overview of the status and properties of the Greenplum Database clusters that this instance of the Command Center is managing.

Pivotal Greenplum Database Status

Included in this section is the current State of the Greenplum Database. The condition can be one of the following:

  • Normal: The database is functioning with no major errors or performance issues.
  • Segment Failure(s): The database is in change-tracking mode or resync mode. Overall performance and system reliability is greatly reduced. See the Pivotal Greenplum Database System Administrator Guide for information about resolving this condition.
  • Database Unreachable: The Greenplum Performance Monitor agent cannot connect to the database. The database is likely down. See the Pivotal Greenplum Database System Administrator Guide for troubleshooting information.
  • Suboptimal Segment Configuration: All segments are not running in their preferred roles. That is primaries are running as mirrors and mirrors are running as primaries, resulting in unbalanced processing.

Health

The Health section of the dashboard displays a pie chart that gives you an overview of the overall health of all the components in your Pivotal Greenplum Database system, including hardware, software, and Greenplum Database instances. A color indicator and associated number indicate the number of hosts in the system that are currently in that particular state. Hosts can have the following states:

  • Normal (Green)
  • Warning (Yellow)
  • Error (Red)
  • Unknown (Question mark)
  • Unreachable (Black)

If you click on any area in the pie chart, the Health tab opens. If you click on the segments of the pie chart that represents hosts in a state of normal, the Health tab opens displaying the details of hosts in that state.

If you have multiple Greenplum Database instances, you can create separate instances of Pivotal Greenplum Command Center for them. Each separate Command Center instance operates on a unique port and has its own unique configuration options.

Key Metrics

The main section of the dashboard displays graphs for key system metrics such as CPU, memory, and query metrics.

Clicking More >> in any of these graphs opens the System Metrics tab displaying Detail by Metric information for that graph.

  • CPU. This line graph displays average CPU usage across the entire cluster, over time (this includes Greenplum Database Master and Standby nodes). Hovering the mouse cursor over lines in the graphs displays a small window with percentage of CPU in use at any given time.
  • Queries/Jobs. This graph displays a summary view of active/queued queries over time.
  • Memory. This graph displays the average memory use across the entire cluster over time.

Alerts

This section lists the content of Greenplum Database error messages, the time they were issued, and the severity level.

Dial-Homes

This section is only populated if this Command Center is monitoring a Data Computing Appliance (DCA) environment.

This section lists any dial-home messages that have been sent to EMC support.

Dial-home messages are generated for Data Computing Appliances (DCAs) only.

About Dial-Homes: The EMC Greenplum Data Computing Appliance supports dial-home functionality through the ConnectEMC software. ConnectEMC is a support utility that collects and sends event data - files indicating system errors and other information - from EMC products to EMC Global Services customer support. ConnectEMC events are triggered based on a hardware failure, a fix to a failed hardware component, or a Greenplum Database startup. Once an ConnectEMC event is triggered, an alert is sent to EMC Support.

Dial-home using ConnectEMC functionality is setup during installation. If your system is not configured to use this functionality, contact EMC Global Services.

Disk usage alerts are issued when your disk usage reaches a specified threshold. This threshold is configured with default values during installation (80% & 90%); these values can be modified by EMC Services using the DCA Setup utility.
