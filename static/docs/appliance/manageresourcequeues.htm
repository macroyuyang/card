Manage Resource Queues

To enable Workload Management, you must first create resource queues.

Resource scheduling is enabled by default when you install Greenplum Database. All database roles must be assigned to a resource queue. If an administrator creates a role without explicitly assigning it to a resource queue, the role is assigned to the default resource queue, pg_default.

Greenplum recommends that administrators create resource queues for the various types of workloads in their organization. For example, you may have resource queues for power users, web users, and management reports. You would then set limits on the resource queue based your estimate of how resource-intensive the queries associated with that workload are likely to be.

You can use Commander Center Manage Resource Queues to create or edit resource queues.

Creating a resource queue involves giving it a name and setting either a query cost limit or an active query limit (or both), and optionally a query priority on the resource queue.

Create a new resource queue, or select an existing resource queue and give values to the following parameters:

  • Active Statements. The maximum number of statements that can run concurrently.
  • Priority: This value defines a queue’s priority relative to other queues in terms of available CPU resources. One of MIN, LOW, MEDIUM, HIGH, MAX.
  • Memory Limit. The total amount of memory that all queries submitted through this queue can consume.

After you have created or edited a resource queue, you then assign database roles (users) to the appropriate resource queue. A resource queue can have multiple roles, but a role can have only one assigned resource queue.
